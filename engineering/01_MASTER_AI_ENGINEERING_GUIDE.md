# 1. Purpose

This document is the official engineering handbook for the AI Video Hosting Platform.

Its purpose is to ensure that every AI model (Claude, ChatGPT, Gemini, Qwen, DeepSeek, or any future AI assistant) follows the same architecture, coding standards, development workflow, and engineering principles.

This document is considered the highest authority for all development decisions.

---

# 2. Project Vision

Build a modern, secure, scalable AI-powered video hosting platform where users can:

- Upload videos
- Stream videos
- Share videos
- Manage uploaded content
- Store videos in AWS S3
- Automatically moderate uploaded videos using AI
- Upgrade storage through subscription plans

The platform must look professional and production-ready.

# 6. Git Workflow

Every completed feature must follow:

git add .

git commit -m "Meaningful message"

git push

Never commit unfinished code.

---

# 7. Security Rules

Passwords must always be hashed.

JWT authentication is mandatory.

Secrets must never be hardcoded.

Environment variables must be used.

Input validation is mandatory.

Rate limiting should be implemented.

Uploaded files must be validated.

---

# 8. Coding Philosophy

Write less.

Write cleaner.

Reuse components.

Avoid duplication.

Prefer readability.

---

# 9. AI Development Rules

Every AI must:

Read engineering documents before generating code.

Never modify unrelated files.

Generate only assigned modules.

Explain generated code when necessary.

Stop after completing assigned tasks.

---

# 10. Definition of Done

A feature is complete only when:

- Code works
- No console errors
- No lint errors
- Documentation updated
- Git commit completed
- GitHub updated
